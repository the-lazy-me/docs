# SillyTavern 接入启航AI教程

## 教程说明

SillyTavern 使用 `Chat Completion` 的 `Custom (OpenAI-compatible)` 接入方式。对于启航AI，这意味着走 OpenAI-compatible 自定义端点，而不是单独的 Claude 接入通道。

## 适用版本或适用形态

- SillyTavern 桌面或自托管部署
- 支持 Chat Completion 自定义来源的版本

## 前置条件

- SillyTavern 可用实例
- 启航AI API Key
- 启航AI OpenAI-compatible API 地址

## 官方支持的接入方式

- 在 API 设置里选择 `Chat Completion`
- 将来源设为 `Custom (OpenAI-compatible)`
- 填写自定义 endpoint URL 和 API Key
- 如上游支持 `/v1/models`，可自动拉取模型；否则手动填写模型 ID

## 逐步配置

### 1. 打开 API 设置

1. 打开 SillyTavern
2. 进入设置
3. 打开 API 配置页

### 2. 选择接入类型

1. 选择 `Chat Completion`
2. 将 `Chat Completion Source` 设为 `Custom (OpenAI-compatible)`

### 3. 填写启航AI信息

1. 填写 `Custom endpoint URL`
2. 填写 `API key`
3. 保存配置

### 4. 选择模型

1. 如果客户端能够从 `/v1/models` 拉取模型，直接选择可用模型
2. 如果列表为空，手动输入模型 ID
3. 重新加载后开始对话

## 验证是否接入成功

- 发送一条测试消息
- 如果机器人能正常回复，说明接入成功
- 如果模型不可见，优先检查 endpoint 是否填写到基础地址层级

## 常见问题

**Q: 为什么不写 Claude 单独通道？**

A: 启航AI这里按官方可核实的 OpenAI-compatible 自定义端点来接，不混写另一套接入流程。

**Q: endpoint 要不要写到 `/chat/completions`？**

A: 不要。只填写基础端点即可，必要时在基础地址层级补 `/v1`。

**Q: 模型看不到怎么办？**

A: 先确认上游是否提供 `/v1/models`，没有就手动输入模型 ID。
