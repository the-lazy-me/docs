# OpenCode 接入启航AI教程

## 教程说明

本教程介绍如何在 OpenCode 中接入启航AI。OpenCode 的主流程是先执行 `/connect`，再通过 `opencode.json` 配置 provider，因此不再使用简单的环境变量直连写法作为主教程。

## 适用版本或适用形态

- OpenCode CLI
- 用户级或项目级 `opencode.json`

## 前置条件

- Node.js 和 OpenCode CLI
- 启航AI API Key
- 可编辑 `opencode.json`

## 官方支持的接入方式

OpenCode 使用 provider 配置接入兼容接口。可确认的核心要素包括：

- `/connect`
- `opencode.json`
- `provider`
- `options.baseURL`

## 逐步配置

1. 在 OpenCode 中执行 `/connect` 完成基础连接流程。
2. 打开或创建 `opencode.json`。
3. 在 provider 配置中使用 OpenAI-compatible 对应条目。
4. 填写启航AI的 `baseURL` 和 `apiKey`。
5. 保存配置后重新启动或刷新会话。

示例：

```json
{
  "provider": {
    "name": "@ai-sdk/openai-compatible",
    "options": {
      "baseURL": "https://api.qhaigc.net/v1",
      "apiKey": "your-api-key"
    }
  }
}
```

## 验证是否接入成功

- `/connect` 流程可以完成
- OpenCode 能正常读取 `opencode.json`
- 发起对话时可以收到启航AI的回复

## 常见问题

**Q: 为什么不再主推 `OPENAI_API_KEY` 和 `OPENAI_BASE_URL`？**

A: OpenCode 官方主流程是 provider 配置和 `opencode.json`，不是只靠环境变量。

**Q: 如果上游不支持自动获取模型怎么办？**

A: 先确保 provider 和 `baseURL` 配置正确，再按 OpenCode 的实际 UI 或配置项手动指定模型。
