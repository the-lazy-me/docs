# New API 接入启航AI教程

## 教程说明

New API 是聚合分发网关。本教程只写当前 `Channels` 和渠道管理的真实接入路径，不沿用旧项目名或旧字段名。

## 适用版本或适用形态

- New API 管理后台
- 已部署的聚合分发实例

## 前置条件

- 已部署 New API
- 管理员账号
- 启航AI API Key
- 启航AI API 地址

## 官方支持的接入方式

- 在 `Channels` 或 `渠道管理` 中新增渠道
- 渠道类型选择 OpenAI-compatible 相关类型
- 填写渠道名称、API Key、API 地址、模型、分组、权重和状态
- 通过 New API 自身的 `/v1` 接口向下游分发

## 逐步配置

### 1. 进入渠道管理

1. 登录 New API 管理后台
2. 打开 `Channels`

### 2. 新增渠道

1. 点击 `Add channel` 或 `添加渠道`
2. 选择与你的上游一致的 OpenAI-compatible 渠道类型
3. 填写启航AI信息

| 配置项 | 值 |
|---|---|
| 渠道名称 | 启航AI |
| API Key | 启航AI API Key |
| API 地址 | 启航AI API 地址 |
| 模型 | 需要暴露给下游的模型 ID |
| 分组 | 例如 `default` |
| 权重 | 按你的路由策略设置 |
| 状态 | 启用 |

4. 保存渠道

### 3. 配置模型映射

1. 如果你需要把上游模型映射成统一名称，进入模型映射相关页面
2. 配置映射后保存

### 4. 下游使用

1. 创建下游令牌
2. 将 New API 的 `/v1` 地址和令牌提供给下游
3. 下游按 OpenAI 接口方式请求即可

## 验证是否接入成功

- 渠道状态显示正常
- 通过测试请求成功返回
- 下游调用 `chat/completions` 时能命中启航AI渠道

## 常见问题

**Q: 渠道离线怎么办？**

A: 先检查 API Key、API 地址和网络连通性，再确认渠道类型是否选对。

**Q: 下游为什么没有走到启航AI？**

A: 检查权重、分组和模型映射是否生效。

**Q: 需要把启航AI写成旧项目名吗？**

A: 不需要。本页只按当前 `New API` 和 `Channels` 术语书写。
