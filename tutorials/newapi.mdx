# New API 接入启航AI教程

[New API](https://github.com/Calcy-II/next-chatgpt-api) 是 One-API 的分支版本，AI API 聚合管理分发系统。本教程介绍如何将启航AI接入 New API 作为上游渠道。

## 前置要求

- 已部署 New API（本地/Docker/云服务器）
- 启航AI API Key（[获取地址](https://api.qhaigc.net)）

## 配置步骤

### 1. 登录 New API 管理后台

访问你的 New API 部署地址，使用管理员账号登录。

### 2. 进入渠道管理

在左侧菜单中找到 **渠道管理**（或 **渠道**），点击进入。

### 3. 添加新渠道

点击 **添加渠道** 按钮。

### 4. 填写启航AI配置

| 配置项 | 值 |
|--------|-----|
| 渠道类型 | **OpenAI 兼容** |
| 名称 | 启航AI |
| API Key | 你的启航AI API Key |
| 代理地址 | `https://api.qhaigc.net/v1` |
| 模型 | 支持以下模型：<br>`gpt-4o`<br>`gpt-4o-mini`<br>`claude-3-5-sonnet`<br>`claude-3-haiku`<br>`gemini-1.5-pro` |
| 分组 | 可选，如 `default` |
| 权重 | 1000（默认最高优先级） |
| 状态 | ✅ 启用 |

### 5. 保存配置

点击 **确定** 或 **保存** 按钮。

### 6. 验证渠道

- 在 **渠道管理** 中查看状态是否显示 **正常**
- 可以点击 **测试** 按钮发送测试请求

## 多渠道配置（可选）

如果有多个 API 上游，可以配置多个渠道并设置不同权重：

```
启航AI（权重1000）- 主用渠道
其他API（权重500）- 备用渠道
```

### 权重分配建议

| 场景 | 权重配置 |
|------|---------|
| 仅使用启航AI | 1000 |
| 启航AI为主，其他备用 | 启航AI:1000，其他:100 |
| 负载均衡 | 多个渠道设为相同权重 |

## 分发下游

### 创建令牌

1. 进入 **令牌管理**
2. 点击 **创建令牌**
3. 设置令牌名称和配额
4. 复制生成的 API Key 分发给下游用户

### 下游使用方法

下游用户使用 New API 的地址 + 令牌访问：

```bash
curl https://your-newapi-domain.com/v1/chat/completions \
  -H "Authorization: Bearer sk-xxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "你好"}]
  }'
```

## 高级配置

### 模型映射

如需自定义模型名称，可在 **模型管理** 中配置映射：

| 原始模型 | 映射名称 |
|---------|---------|
| gpt-4o | gpt-4o |
| claude-3-5-sonnet | claude-sonnet |

### 渠道健康检查

New API 支持自动健康检查，可在设置中配置检查间隔和失败阈值。

## 常见问题

**Q: 渠道显示离线怎么办？**

A: 检查：
- API Key 是否正确
- 代理地址是否包含 `/v1`
- 网络是否能访问 `api.qhaigc.net`

**Q: 如何查看渠道用量？**

A: 在 **渠道管理** 中点击对应渠道，查看请求统计和用量数据。

**Q: 下游用户如何使用特定模型？**

A: 在请求中指定模型名称即可，New API 会自动路由到启航AI渠道。
