# Cherry Studio 接入启航AI教程

## 教程说明

Cherry Studio 通过自定义 provider 接入启航AI。官方当前要求在 `Model Services` 中新增 provider，再在 `Model management` 里手动维护模型列表。

## 适用版本或适用形态

- Cherry Studio 桌面端
- 已支持自定义模型服务的版本

## 前置条件

- Cherry Studio 客户端
- 启航AI API Key
- 启航AI可访问的 API 地址

## 官方支持的接入方式

- 在 `Model Services` 中新增自定义 provider
- `Provider Type` 选择 `OpenAI`
- 在 `Model management` 中手动添加模型 ID
- 使用 API Key 右侧的校验能力确认连通性

## 逐步配置

### 1. 打开模型服务设置

1. 打开 Cherry Studio
2. 进入设置
3. 打开 `Model Services`

### 2. 新增 provider

1. 点击 `Add` 或 `+`
2. 填写 `Provider Name`
3. 将 `Provider Type` 设为 `OpenAI`

### 3. 填写启航AI信息

1. 填写 `API Key`
2. 填写 `API address`
3. 如界面显示 `Model management`，先保留为空，后续单独维护模型 ID
4. 保存 provider

### 4. 维护模型列表

1. 进入 `Model management`
2. 手动添加需要使用的模型 ID
3. 保存后回到对话页选择该 provider

## 验证是否接入成功

- 点击 API Key 右侧的校验按钮确认配置可用
- 发送一条测试对话，确认能够返回启航AI回复
- 如果模型列表为空，先检查是否已在 `Model management` 中手动添加

## 常见问题

**Q: 为什么模型不会自动出现？**

A: Cherry Studio 这类自定义 provider 通常要在 `Model management` 里手动维护模型 ID。

**Q: 连接测试失败怎么办？**

A: 检查 API Key、API address 和 provider 类型是否都按 `OpenAI` 配置。

**Q: 为什么看不到同一套模型？**

A: 不同版本的界面文案可能略有差异，但核心仍是自定义 provider 和手动维护模型列表。
