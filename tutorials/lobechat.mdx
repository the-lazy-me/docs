# LobeChat 接入启航AI教程

## 教程说明

本教程区分两种场景：在线版或桌面端的用户侧配置，以及自托管场景的环境变量配置。前者走 LobeChat 的 OpenAI provider，后者走官方支持的自托管变量。

## 适用版本或适用形态

- LobeChat 在线版
- LobeChat 桌面端或网页端
- LobeChat 自托管部署

## 前置条件

- 可访问的 LobeChat
- 启航AI API Key
- 自托管场景下可修改部署环境变量

## 官方支持的接入方式

- 在线版或桌面端：在设置中选择 `OpenAI` provider，填写 API Key
- 自托管：配置 `OPENAI_API_KEY` 和 `OPENAI_PROXY_URL`
- 如果你不是自托管，不要假设一定存在任意自定义 Base URL 输入框

## 逐步配置

### 1. 在线版或桌面端

1. 打开 LobeChat 设置
2. 进入语言模型或 `Model` 设置页
3. 选择 `OpenAI` provider
4. 填写启航AI API Key
5. 保存配置
6. 返回对话页并选择可用模型

### 2. 自托管部署

1. 在部署环境中设置 `OPENAI_API_KEY`
2. 设置 `OPENAI_PROXY_URL=https://api.qhaigc.net/v1`
3. 重启服务或重新部署
4. 打开 LobeChat 并进入模型设置页确认配置已生效

## 验证是否接入成功

- 新建一条对话并发送消息
- 如果能收到启航AI回复，说明接入成功
- 如果自托管场景仍不可用，优先检查环境变量是否被实例实际读取

## 常见问题

**Q: 为什么在线版没有自定义 Base URL？**

A: 本页不把在线版写成可任意填网关地址的通用接入流程。若你是自托管，再使用 `OPENAI_PROXY_URL`。

**Q: 模型列表为空怎么办？**

A: 先确认 `OpenAI` provider 已保存并刷新页面，再检查上游是否可用。

**Q: 连接失败怎么办？**

A: 检查 API Key、网络可达性，以及自托管环境变量是否已生效。
