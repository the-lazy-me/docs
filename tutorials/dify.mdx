# Dify 接入启航AI教程

## 教程说明

[Dify](https://dify.ai) 的模型接入已经转向模型提供商插件体系。本教程只写当前可核实的接入方式：通过已安装的模型 provider 插件接入 OpenAI-compatible 服务。

## 适用形态

- Dify 云版
- Dify 自托管实例
- 已启用对应模型 provider 插件的环境

## 前置条件

- 可访问的 Dify 控制台
- 启航AI API Key
- 一个可用的 OpenAI-compatible provider 插件，或 Dify 当前版本中对应的 OpenAI 模型提供商插件

## 官方支持的接入方式

Dify 的模型配置以 provider 插件为中心，而不是固定的旧式“模型供应商”页面。官方文档提到的模型设计规则中包含 `fetch-from-remote` 模式，适合通过 provider 凭证统一拉取模型。

## 逐步配置

### 1. 打开模型提供商配置

进入 Dify 控制台，找到模型提供商或模型设置入口，添加对应的 OpenAI 模型 provider 插件。

### 2. 填写 provider 凭证

按当前插件页面要求填写启航AI的 API Key，以及插件要求的 API 地址或 Base URL。

### 3. 拉取或选择模型

如果插件支持远程拉取模型，先同步模型列表；如果需要手动配置模型，则按插件页面要求填写模型 ID。

### 4. 在应用中选择该模型

在聊天应用、Agent 或工作流的模型选择位置，选择刚才配置好的 provider 模型。

### 5. 保存并测试

保存配置后，用一个最小请求验证应用是否能正常生成回复。

## 验证是否接入成功

- 模型提供商页面能正常保存
- 应用侧可以选到该模型
- 测试对话能返回正常响应

## 常见问题

**Q: 找不到“OpenAI Compatible”入口怎么办？**

A: Dify 当前是插件化 provider 架构，不同版本的入口名称可能不同。优先查找模型提供商插件，而不是沿用旧版菜单名称。

**Q: 为什么不直接写固定模型列表？**

A: 具体可见模型取决于 provider 插件和当前版本配置，若无强证据不写死。

**Q: 需要把 Base URL 一定写成 `/v1` 吗？**

A: 只在当前插件或官方文档明确要求时这样写；否则按插件字段要求填写 API 地址即可。
