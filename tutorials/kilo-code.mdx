# Kilo Code 接入启航AI教程

[Kilo Code](https://github.com/kilo-ai/kilo-code) 是 VS Code AI 编程插件，Cline 的另一分支。本教程介绍如何接入启航AI。

## 前置要求

- VS Code
- Kilo Code 插件
- 启航AI API Key（[获取地址](https://api.qhaigc.net)）

## 安装 Kilo Code

### 从 VS Code 扩展市场安装

1. 打开 VS Code
2. 点击扩展图标（`Ctrl/Cmd + Shift + X`）
3. 搜索 **Kilo Code**
4. 点击 **安装**

## 配置步骤

### 1. 打开 Kilo Code 面板

安装完成后，点击左侧 Kilo Code 图标。

### 2. 进入设置

点击面板右上角的 **设置** 图标（齿轮）。

### 3. 选择 API Provider

在 **API Provider** 下拉框中选择 **OpenAI Compatible**。

### 4. 填写配置

| 配置项 | 值 |
|--------|-----|
| API Key | 你的启航AI API Key |
| Base URL | `https://api.qhaigc.net/v1` |
| Model | 选择模型，如 `gpt-4o` |

### 5. 选择模型

在 **Model** 下拉框中选择启航AI提供的模型：

- `gpt-4o`
- `gpt-4o-mini`
- `claude-3-5-sonnet`
- `claude-3-haiku`

### 6. 保存并开始使用

关闭设置面板，即可开始使用。

## 与 Cline/Roo Code 的区别

## 企业功能

### 团队共享配置

Kilo Code 支持团队共享 API 配置：

```json
// .kilo/config.json
{
  "apiProvider": "openai-compatible",
  "apiBase": "https://api.qhaigc.net/v1",
  "defaultModel": "gpt-4o"
}
```

### 审计日志

启用审计日志记录所有 AI 交互：

```json
{
  "auditLog": true,
  "auditLogPath": "./logs/kilo-audit.log"
}
```

### 代码审查工作流

集成到 CI/CD 流程：

```yaml
# .github/workflows/kilo-review.yml
- name: Kilo Code Review
  run: kilo review --model claude-3-5-sonnet
```

| `Ctrl/Cmd + Enter` | 发送消息 |

## 常见问题

**Q: 与 Cline/Roo Code 冲突？**

A: 建议只安装其中一个插件。

**Q: 企业功能需要付费？**

A: 部分企业功能需要 Kilo Code Enterprise 许可证。

**Q: 如何查看 Token 消耗？**

A: 登录 [启航AI控制台](https://api.qhaigc.net) 查看。
