# LangBot 接入启航AI教程

## 教程说明

LangBot 当前更适合按管理界面的 `Configure Models` / `Models` 页面接入模型，而不是把旧版配置文件当作主路径。启航AI在这里按模型项逐个添加。

## 适用版本或适用形态

- LangBot 控制台或管理面板
- 支持模型页面配置的版本

## 前置条件

- LangBot 已部署
- 可访问管理界面
- 启航AI API Key
- 启航AI API Request URL

## 官方支持的接入方式

- 在 `Configure Models` 或 `Models` 页面新增模型
- 填写 `Model Name`
- 选择 `Model Provider`
- 填写 `Request URL`
- 填写 `API Key`
- 知识库场景需要单独配置 embedding 模型

## 逐步配置

### 1. 打开模型配置页

1. 进入 LangBot 管理界面
2. 打开 `Configure Models` 或 `Models`

### 2. 新增 LLM 模型

1. 点击新增模型
2. 填写 `Model Name`
3. 选择 `Model Provider`
4. 填写 `Request URL`
5. 填写 `API Key`
6. 保存模型

### 3. 选择默认模型

1. 在 pipeline 或机器人配置中选择刚添加的模型
2. 保存并重新加载配置

### 4. 如果使用知识库

1. 额外配置 embedding 模型
2. 为 embedding 单独填写 Request URL、API Key 和模型名

## 验证是否接入成功

- 在绑定的聊天平台发送一条消息
- 如果 Bot 正常回复，说明模型接入成功
- 如果知识库查询异常，先检查 embedding 模型是否单独配置

## 常见问题

**Q: 为什么不再主写旧的 config.yaml？**

A: 当前教程以官方 UI 的模型配置页为主，避免把旧部署模板写成唯一入口。

**Q: 只配 LLM 可以吗？**

A: 可以用于普通对话，但知识库场景还要单独配置 embedding 模型。

**Q: 连接失败怎么办？**

A: 检查 Request URL、API Key 和模型名称是否填写正确。
