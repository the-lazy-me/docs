# AstrBot 接入启航AI教程

## 教程说明

AstrBot 当前更适合按控制台里的 `Service Providers` 流程接入模型服务。启航AI在这里按 `OpenAI` provider 配置，再把模型拉取并启用。

## 适用版本或适用形态

- AstrBot 控制台管理模式
- 已可访问 AstrBot Console 的部署

## 前置条件

- AstrBot 已部署并可访问控制台
- 启航AI API Key
- 启航AI API Base URL

## 官方支持的接入方式

- 在 `AstrBot Console` 的 `Service Providers` 中添加 `OpenAI` provider
- 填写 `API Key` 和 `API Base URL`
- 使用 `Get Model List` 拉取模型
- 在 `Configuration` 页面选择默认对话模型

## 逐步配置

### 1. 打开服务提供商页面

1. 进入 AstrBot Console
2. 打开 `Service Providers`

### 2. 新增 provider

1. 点击 `Add Provider`
2. 选择 `OpenAI`
3. 填写 `API Key`
4. 填写 `API Base URL`
5. 保存 provider

### 3. 拉取并启用模型

1. 点击 `Get Model List`
2. 选择需要使用的模型
3. 启用该模型

### 4. 选择默认模型

1. 打开 `Configuration`
2. 选择默认对话模型
3. 保存配置

## 验证是否接入成功

- 在 QQ、微信或 Telegram 中发送一条消息
- 如果 AstrBot 正常回复，说明接入成功
- 如果无回复，优先检查 provider 状态和默认模型是否已启用

## 常见问题

**Q: 为什么不再主写 config.yaml？**

A: 当前官方主线是控制台里的 provider 管理流程，不再建议把旧 YAML 当成主教程。

**Q: 模型拉不下来怎么办？**

A: 检查 API Key、Base URL 和网络连通性，再确认 provider 是否保存成功。

**Q: Bot 起来了但不回复怎么办？**

A: 先确认 `Configuration` 里已选择默认模型，再检查对应平台适配器是否已绑定。
